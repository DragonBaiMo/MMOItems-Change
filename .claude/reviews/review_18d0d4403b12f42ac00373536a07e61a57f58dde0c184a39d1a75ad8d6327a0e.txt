# 变更摘要：引入 DurabilityResult 结果模式重构耐久度系统，提供更安全的库存更新与状态反馈机制。

## 文件：MMOItems-API/src/main/java/net/Indyuce/mmoitems/api/interaction/util/DurabilityItem.java
### L113：[MEDIUM] buildResult() 中的副作用会导致音效重复
在 `buildResult()` 中，当物品损坏且 `isLostWhenBroken()` 为真时，会播放 `ENTITY_ITEM_BREAK` 音效（仅限 `maxDurability == 0` 的物品）。
然而，调用方（如 `DurabilityListener.java` L125）在检测到 `result.isBroken()` 后也会播放同一个音效。
**后果：** 对于自定义材质物品（如 Stick），玩家会听到双重破碎音效（Double Sound Effect）。
**建议：** `buildResult()` 应保持纯粹的状态计算，将音效播放等副作用交由调用方处理。建议移除此处的音效播放逻辑。

```diff
             // 损坏后丢失
             if (isLostWhenBroken()) {
                 // 播放破碎音效
                 if (player != null) {
-                    if (item.getType().getMaxDurability() == 0) {
-                        player.getWorld().playSound(player.getLocation(), Sounds.ENTITY_ITEM_BREAK, 1, 1);
-                    }
                     PlayerData.get(player).getInventory().watchVanillaSlot(
                             io.lumine.mythic.lib.api.player.EquipmentSlot.fromBukkit(slot), Optional.empty());
                 }
```
